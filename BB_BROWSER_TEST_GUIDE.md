## 💡 常见问题

### Q1: bb-browser 和 OpenClaw 的关系？

**A**:
- bb-browser 是一个命令行工具
- OpenClaw 是一个浏览器自动化框架
- bb-browser 可以使用 OpenClaw 的浏览器（通过 `--openclaw` 参数）
- 但也可以使用本地 Chrome 浏览器（不依赖 OpenClaw）

---

### Q2: 为什么不用 `--openclaw` 参数？

**A**:
- OpenClaw Gateway 目前无法启动
- 使用本地浏览器模式更简单
- 本地浏览器模式已经可以正常工作

---

### Q3: 如何测试会话保持？

**A**:
1. 使用 bb-browser 登录 YP 平台
2. 关闭所有 bb-browser 相关进程
3. 等待一段时间（1小时、24小时等）
4. 重新使用 bb-browser 访问受保护页面
5. 观察是否需要重新登录

---

### Q4: 如果登录失败怎么办？

**A**:
1. 检查用户名和密码是否正确
2. 检查验证码是否输入正确
3. 查看浏览器控制台是否有错误信息
4. 使用 `bb-browser snapshot` 查看页面状态
5. 使用 `bb-browser errors` 查看 JavaScript 错误
