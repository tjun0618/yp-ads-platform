## 二、发现的问题

### 问题 1：API 采集混入非美国站商品（严重）

**现象**: YP API 返回的 `price` 字段包含多个 Amazon 站点的商品，同一 ASIN 可能有多条记录。

**实际案例** (ASIN=B07FRRHPJD):
| Product ID | 站点 | 价格 | 佣金 | 商品名语言 |
|-----------|------|------|------|-----------|
| 543934 | 美国站 | USD 69.99 | 7.5% | 英语 |
| 1333492 | 欧洲站 | EUR 28.44 | 6% | 德语 |
| 438891 | 加拿大站 | CAD 86.99 | 3.75% | 英语 |

**影响**: 飞书 8,401 条中，**无法确定有多少是真正美国站商品**。采集时 price 字段被处理为纯数字，丢失了货币前缀（USD/EUR/CAD），现在已无法区分。

**价格分布佐证**: 中位数 $29.69、均价 $49.79，价格从 $0.01 到 $3,999。其中有 1,107 条价格=0。如果只看美国站商品，价格区间和分布可能完全不同。

### 问题 2：商品归属错误（严重）

**现象**: API 返回的商品所归属的商户可能不正确。网页抓取到的映射也会把商品归到错误商户下。

**实际案例**:
- NORTIV 8 品牌页显示 B07FRRHPJD 属于自己（佣金 7.5%）
- 但 API 和网页抓取都把它归到 DREAM PAIRS（佣金 6%）
- **实际检查**: DREAM PAIRS 的商户页面（MID:362851）上根本没有这个 ASIN

**飞书数据统计**: 在 214 条有链接的记录中，22.4%（48条）的商品名与标注的商户名不匹配。典型案例：
- 标注商户 DREAM PAIRS，商品名却是 NORTIV 8（德语）
- 标注商户 HOFFREE，商品名却是 Adirondack Chair（属于另一个品牌）
- 标注商户 MECO ELEVERDE，商品名却是 OCOOPA（不同品牌）

**根因**: YP 平台的商品-商户关系是通过 API 接口返回的，但 API 的归属逻辑不可靠。可能原因是 YP 平台内部的数据同步或分类逻辑问题。

### 问题 3：投放链接覆盖极低（严重）

**飞书表**: 8,401 条中仅 214 条有 Tracking URL（2.5%），意味着 97.5% 的商品无法直接用于 Google Ads 投放。

**asin_map**: 96,794 条中有 53,165 条有链接（54.9%），但 **与飞书交叉覆盖率极低**。

**NORTIV 8 Excel**: 5,461 条全部有链接——这才是"已申请通过"品牌应有的数据完整度。

**对比**: NORTIV 8 一个品牌就有 5,461 条可用商品。网页抓取 9,679 个 ASIN 但只匹配到 60 个 NORTIV 8 的商品（1.1%）。如果每个获批品牌都有类似规模，当前采集体系漏掉了绝大多数可用商品。

### 问题 4：同一 ASIN 重复记录

**飞书**: 8,401 条中只有 8,332 个唯一 ASIN，63 个 ASIN 有重复（最多一个 ASIN 出现 3 次）。

**asin_map**: 结构上每个 ASIN 只映射一个商户，但如问题2所述，这个映射可能是错的。

### 问题 5：推广状态更新未完成

| 状态 | 数量 | 占比 |
|------|------|------|
| 未匹配 | 4,650 | 55.4% |
| 空（未标注） | 3,110 | 37.0% |
| 待申请 | 426 | 5.1% |
| 可推广 | 215 | 2.6% |

3,110 条（37%）因为 token 过期未能标注状态。

### 问题 6：322 条商品已下架

飞书中有 322 条 Product Status = "Offline"，不应该出现在投放数据中。

## 三、问题根因分析

### 根因 1：YP API 数据质量问题

YP Offers API 返回的是**全球多站点**的商品数据，没有货币标识字段和站点标识字段。采集脚本无法区分美国站/欧洲站/加拿大站商品，导致：
- 价格失去货币上下文
- 佣金率对应不同站点
- 投放链接可能指向错误站点

### 根因 2：商品-商户归属不可靠

YP 平台内部的商品归属关系与实际品牌页面不一致。API 返回的 merchant_id/merchant_name 不一定正确。网页抓取也只能获取到 API 已经错误归属的数据。

### 根因 3：采集策略效率低

当前方案是"从 API 全量采集商品 → 逐商户网页抓取投放链接 → 匹配到飞书"。但：
- API 全量约 76 万条，每轮 1000 条，需要 760 轮才能采完
- 网页抓取每个商户最多 30 条商品，NORTIV 8 有 5,461 条但只抓到 30 条
- 飞书表 8,401 条仅占 API 总量的 1.1%

### 根因 4：缺少数据清洗步骤

采集后直接写入飞书，没有做去重、货币过滤、商户验证、链接有效性检查等清洗步骤。

## 四、三种数据源对比

| 维度 | API 采集 | 网页抓取 | Excel 下载 |
|------|---------|---------|-----------|
| **数据完整性** | 全平台（全球站点） | 仅已抓取的商户 | 单品牌全量 |
| **价格准确性** | 有但无货币标识 | 无价格 | 准确（USD） |
| **佣金准确性** | 有但可能对应错站点 | 有（从页面获取） | 准确（USD 对应） |
| **投放链接** | API 不提供 | 需逐商户抓取 | 全部有 |
| **商户归属** | **不可靠** | 依赖 API 数据 | **准确**（从品牌页） |
| **覆盖规模** | ~76 万条 | ~9.6 万 ASIN | 单品牌数千条 |
| **操作成本** | 自动，但需大量翻页 | 自动，但耗时长 | 手动下载，秒级完成 |

## 五、修复方案建议

### 方案：以"品牌页 Excel 导入"为主，API 降级为"商品发现"

**核心思路**: 不再通过 API 或网页抓取获取可投放数据。改为：
1. 你在 YP 网页端申请品牌 → 审批通过
2. 从品牌页面 Download Products 导出 Excel
3. 用脚本将 Excel 导入飞书，自动标注"可推广"

**优点**:
- 数据 100% 准确（价格、佣金、链接都来自品牌页）
- 100% 有投放链接
- 只包含美国站商品
- 操作极快（下载 + 导入秒级完成）

**需要开发的工具**:
1. **Excel 导入飞书脚本** — 将品牌页下载的 Excel 直接导入飞书 Offers 表，自动匹配字段
2. **商品去重逻辑** — 如果某个 ASIN 已在飞书表中（来自其他品牌的 Excel），比较佣金率，保留佣金高的
3. **货币/站点过滤** — 确保只导入 USD 价格的商品

**API 的保留用途**:
- 发现新品牌和新商品（用于决定申请哪些品牌）
- 获取商品评分、评论数等 Amazon 数据（Excel 可能不含这些）

### 飞书表清理

在启动新方案前，建议：
1. 新建一个干净的"可推广商品"表，只从品牌 Excel 导入
2. 现有 8,401 条保留但标记为"旧数据-需验证"，不用于实际投放
3. 清除 322 条 Offline 商品
4. 补完 3,110 条未标注推广状态的记录
