# Monetization API 数据采集指南

## 概述

**重要发现**：经过详细分析 YP 平台的 API 文档，我发现 **Monetization API 和 Transaction API 实际上是同一个 API**，它们的端点、参数和返回数据完全相同！

这意味着 **Monetization API 并不能获取您需要的品牌和商品信息**。

---

## Monetization API 能采集的数据

### 1. API 基本信息

| 参数 | 值 |
|-----|---|
| **URL** | `https://yeahpromos.com/index/Getorder/getorder` |
| **返回格式** | JSON |
| **HTTP 方法** | GET |
| **请求限制** | 每分钟 10 次 |

### 2. 能采集到的数据字段

| 字段名 | 说明 | 示例 | 是否包含品牌信息 |
|--------|------|------|----------------|
| **PageTotal** | 总页数 | `5` | ❌ 否 |
| **PageNow** | 当前页码 | `1` | ❌ 否 |
| **Num** | 销售交易总数 | `100` | ❌ 否 |
| **Limit** | 每次输出的最大数量 | `1000` | ❌ 否 |
| **id** | 记录 ID | `12345` | ❌ 否 |
| **advert_id** | 广告 ID | `ad_001` | ⚠️ 可能相关 |
| **oid** | 订单 ID | `ORDER123456` | ❌ 否 |
| **creationDate_time** | 交易时间 | `2024-01-15 10:30:00` | ❌ 否 |
| **amount** | 销售金额 | `99.99` | ❌ 否 |
| **sale_comm** | 销售佣金 | `9.99` | ❌ 否 |
| **status** | 佣金状态 | `Approved` | ❌ 否 |
| **tag1** | 用户标签 1 | `campaign1` | ❌ 否 |
| **tag2** | 用户标签 2 | `source1` | ❌ 否 |
| **tag3** | 用户标签 3 | `custom` | ❌ 否 |

### 3. 数据说明

#### 3.1 这是交易数据，不是品牌/商品数据

**Monetization API 获取的是**：
- ✅ 交易记录（已完成的订单）
- ✅ 佣金数据（已赚取的佣金）
- ✅ 交易状态（订单状态）
- ✅ 订单信息（订单 ID、金额等）

**Monetization API 无法获取**：
- ❌ 品牌列表（Merchants）
- ❌ 商品列表（Products）
- ❌ 品牌的原始链接
- ❌ 商品的原始链接
- ❌ 商品价格（实时价格）
- ❌ 商品佣金率（实时佣金）
- ❌ 商品图片
- ❌ 商品描述
- ❌ 商品评分和评论

#### 3.2 advert_id 的局限性

虽然返回的 `advert_id` 字段可能与品牌相关，但：
- ⚠️ 只能获取到您有交易记录的品牌
- ⚠️ 无法获取完整的品牌列表
- ⚠️ 无法获取品牌的详细信息（名称、佣金率等）
- ⚠️ 无法获取该品牌下的所有商品

## 为什么 Monetization API 无法满足您的需求？

### 核心原因

1. **API 类型不同**
   - Monetization API = 交易数据 API
   - 您需要 = 品牌/商品数据 API

2. **数据用途不同**
   - Monetization API：用于查看已完成的订单和佣金
   - 您需要：用于查看所有可推广的品牌和商品

3. **数据范围不同**
   - Monetization API：只有您有交易记录的品牌和商品
   - 您需要：所有可用的品牌和商品

### 类比说明

- **Monetization API** 就像"银行账户流水"——只显示您已经赚到的钱
- **您需要的数据** 就像"商品目录"——显示所有可以购买的商品

## 正确的解决方案

### 方案 1：检查 YP 平台是否有品牌/商品 API ⭐⭐⭐⭐⭐

**需要确认的问题**：
- YP 平台是否提供了获取品牌列表的 API？
- YP 平台是否提供了获取商品列表的 API？
- 如果有，API 端点是什么？

**如何确认**：
1. 访问 YP Tools 页面：https://www.yeahpromos.com/index/tools/select
2. 查看是否有 "Merchants API" 或 "Products API"
3. 查看 API 文档，确认端点和参数

### 方案 2：使用浏览器自动化 ⭐⭐⭐⭐

**步骤**：
1. 登录 YP 平台
2. 访问 Merchants 页面，提取品牌列表
3. 访问 Products 页面，提取商品列表
4. 提取所有需要的数据字段
5. 导出为 CSV 或 JSON
6. 上传到飞书

**优点**：
- 可以获取所有品牌和商品
- 数据完整

**缺点**：
- 速度较慢
- 需要维护

### 方案 3：使用现有采集脚本 ⭐⭐⭐⭐⭐

**已存在的脚本**：
- `collect_merchants_and_products.py` - 品牌和商品采集脚本
- `scrape_amazon_products_improved.py` - 亚马逊商品采集脚本

**优点**：
- 已经创建好
- 已经验证可行
- 可以直接使用
