# 后置质量检查模块 (Post-Generation QA)

> 版本: v1.0 | 2026-03-29
> 用途: 广告方案生成完成后，强制执行质量检查，不合格项必须修复后才能输出

---

## 一、检查流程

```
广告方案生成完成
    ↓
执行 6 项 QA 检查
    ↓
有不合格项？ → 标记问题 → 修复 → 重新检查
    ↓
全部通过 → 输出最终方案
```

---

## 二、QA 检查项

### QA-1: 价格一致性检查

```
检查内容:
- 扫描所有标题和描述中的价格引用
- 与输入的商品价格对比
- 检查是否有$0.00、空价格、或与输入不一致的价格

自动修复:
- $0.00 → 替换为实际价格
- 错误价格 → 替换为实际价格
- 缺少价格的关键位置 → 补充实际价格

严重级别: 🔴 致命（必须修复）
```

### QA-2: 广告组重复检测

```
检查内容:
- 比较所有广告组之间的关键词集合
- 比较所有广告组之间的标题集合
- 比较所有广告组之间的描述集合
- 计算 Jaccard 相似度

判定标准:
- 关键词重复率 > 30% → 🔴 合并或删除
- 标题重复率 > 20% → 🟡 替换为差异化文案
- 描述重复率 > 20% → 🟡 替换为差异化文案

自动修复:
- 完全相同的广告组 → 删除其中一个
- 关键词重复 → 保留在主题更匹配的组
- 文案重复 → 基于该组主题重新生成差异化文案

严重级别: 🔴 高（严重影响投放效率）
```

### QA-3: 关键词真实性检查

```
检查内容:
- 逐个检查关键词是否是"真实用户会搜索的词"
- 检测模板残留模式

检测规则（命中任一条即标记为可疑）:
1. 包含 "that works" → 虚假关键词
2. 包含 "fix {brand}" → 负面意图或虚假
3. 包含 "need a {product}" → 不自然
4. 包含 "{brand} {product} features" → 无搜索量（除知名品牌）
5. 包含 "{brand} {product} how it works" → 不适用（非电子/软件产品）
6. 超过6个词 → 可能太长，需人工确认
7. 完全由品牌词+泛描述组成 → "best nayno products" 对未知品牌无价值

判定标准:
- 单个广告组可疑关键词 > 2个 → 🟡 警告，建议替换
- 全方案可疑关键词 > 5个 → 🔴 必须修复

自动修复:
- 删除所有命中规则的关键词
- 用品类核心词替换（参考keyword-engine.md）

严重级别: 🔴 高（无效关键词直接浪费预算）
```

### QA-4: 模板残留检测

```
检查内容:
- 检测标题和描述中未替换的模板占位符

检测规则:
1. 包含 "solves your problem" 且前面无具体问题名词 → 模板残留
2. 包含 "premium feature" / "advanced design" / "built for performance" → 空泛模板
3. 包含 "your {product}" / "your {problem}" → 未替换占位符
4. 描述中缺少任何产品具体信息（品类名、功能、材质、尺寸） → 空泛模板
5. 两个完全相同的广告组 → 复制粘贴错误

判定标准:
- 发现任何模板残留 → 🔴 必须修复

自动修复:
- 占位符 → 用实际产品信息替换
- 空泛描述 → 用产品具体信息重写
- 复制广告组 → 删除重复，重新生成

严重级别: 🔴 致命（用户看到空话会直接跳出）
```

### QA-5: 否定关键词适配性检查

```
检查内容:
- 账户级否定词中是否有品类不相关的词
- 是否误杀了高价值流量

检测规则:
1. 首饰盒产品出现 -ingredients / -nutrition / -side effects → 品类不匹配
2. 宠物清洁产品出现 -"how to" / -guide → 可能误杀高价值流量
3. 礼品型产品出现 -review → 误杀决策流量
4. 任何产品出现 -problem / -issue（功能型产品除外） → 需要确认

判定标准:
- 品类不匹配否定词 > 2个 → 🟡 建议清理
- 误杀高价值流量 > 1个 → 🔴 必须修复

严重级别: 🟡 中（影响流量质量但不致命）
```

### QA-6: 字符限制与格式检查

```
检查内容:
- 所有标题 ≤ 30字符（含空格）
- 所有描述 ≤ 90字符（含空格）
- 字符数标注是否准确
- 否定关键词格式是否正确（-前缀，多词组用引号）

判定标准:
- 超字符 → 🔴 必须修复（Google会拒审或截断）
- 字符数标注不准确 → 🟡 建议修正
- 否定词格式错误 → 🟡 建议修正

严重级别: 🔴 致命（超字符广告会被拒审）
```

```

  广告方案质量检查报告
═══════════════════════════════════════

QA-1 价格一致性: ✅ 通过 / ❌ 发现 X 处错误
  → [具体错误位置和修复建议]

QA-2 广告组重复: ✅ 通过 / ⚠️ 发现 X 处重复
  → [具体重复位置和修复建议]

QA-3 关键词真实性: ✅ 通过 / ❌ 发现 X 个可疑关键词
  → [可疑关键词列表和替换建议]

QA-4 模板残留: ✅ 通过 / ❌ 发现 X 处残留
  → [残留位置和修复建议]

QA-5 否定词适配性: ✅ 通过 / ⚠️ 发现 X 处不匹配
  → [不匹配列表和建议]

QA-6 字符与格式: ✅ 通过 / ❌ 发现 X 处超限
  → [超限列表和修复建议]

═══════════════════════════════════════
  总评: ✅ 全部通过 / ❌ X 项需修复
═══════════════════════════════════════
```

## 四、可执行性评估

在QA通过后，额外执行可执行性评估:

```
评估维度:

1. 盈利可行性 (Pass/Fail)
   - 目标CPA是否合理？（对比佣金和市场CPC）
   - 如果单次佣金 < $1.5 → 输出可行性警告
   - 如果目标CPA < 行业最低CPC的50% → 输出严重警告

2. 预算合理性 (Pass/Fail)
   - 广告系列数量是否匹配预算规模？
   - 低预算(<$15/天) → 不超过2个Campaign
   - 中预算($15-50/天) → 2-3个Campaign
   - 高预算(>$50/天) → 3-5个Campaign

3. 关键词覆盖度 (Pass/Fail)
   - 是否覆盖了品类核心词？
   - 是否覆盖了主要使用场景？
   - 是否覆盖了购买意图词？

4. 竞争力评估 (1-5星)
   - 文案是否有差异化卖点？
   - 是否利用了产品的独特优势？
   - 价格展示是否有吸引力？

输出:
  如果任何维度 Fail → 在方案末尾输出明确的改进建议
  如果盈利可行性 Fail → 建议考虑其他营销渠道
```
