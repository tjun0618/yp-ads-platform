# YP 平台投放链接采集 - 2026-03-23 工作总结

## 📋 任务目标
获取飞书 Offers 表中 **3,569 个商品的 YP 平台投放链接**（Tracking URLs），用于 Google Ads 推广。

---

## 🔍 核心问题分析

### 问题描述
- 飞书 Offers 表有 3,569 个亚马逊商品（ASIN）
- YP API (`getoffer` 接口) 的 `tracking_url` 字段为空/null
- 需要通过 YP 网页端获取 tracking URL

### 两个可行方案的评估

#### 方案 1：品牌名匹配（快速但不可靠）
- **方法**：从商品名提取第一个词作为品牌，在 10,020 个商户中做模糊匹配
- **优势**：速度快，几分钟即可完成
- **劣势**：精度低（仅 2.8% 覆盖率）
- **问题**：
  - 亚马逊商品名第一个词是产品描述词，不是品牌名（如 "Magnetic Wireless" 不对应任何商户）
  - YP 商户名与亚马逊品牌名维度不同（如商户 "Amazon US" 卖的品牌是数百个不同的牌子）
  - 品牌匹配结果大量误匹配（"DREAM" → parfumdreams 香水网站）

**结论：此方案不可行** ❌

#### 方案 2：全量网页抓取（慢但可靠）
- **方法**：遍历 10,020 个商户，通过 Cookie 访问商户页面，解析 HTML 获取 ASIN 和 tracking URL
- **优势**：
  - 精度 100%（直接从 YP 网页获取）
  - 获得完整的商品-商户-投放链接映射
  - 无误匹配风险
- **缺点**：耗时较长（需数小时）

**结论：此方案是唯一可靠的路径** ✅

## 🎯 关键学到

### 数据集成中的常见陷阱
1. **API 限制**：YP getoffer 接口每页最多 100 条，全量 76 万条需 79 天
   - **解决方案**：改用网页端抓取（绕过 API 限制）

2. **字段缺失**：API 返回的 `tracking_url` 为 null
   - **根因**：YP 平台需要先申请商户才会分配 track token
   - **解决方案**：从 HTML 页面直接提取

3. **品牌匹配精度低**：商品名 ≠ 商户名 ≠ 品牌名
   - **教训**：不同数据源之间的关键字段需要精确匹配，模糊匹配易误触发

### 性能优化
- 网页抓取 10,020 个商户耗时 ~5 小时，性能可接受
- 关键瓶颈：HTTP 请求超时（部分商户页面响应慢）
- 可优化：并发请求（当前串行）、增加重试机制
