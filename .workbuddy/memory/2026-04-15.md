# 2026-04-15 工作日志

## 商户作战室 - 广告流程页面

### 功能概述
完成"广告流程"页面（`/workflow`）的完整功能实现，实现从商户ID到广告生成的自动化流程。

### 已完成功能

#### 1. 流程图页面基础
- 9步广告生成流程可视化
- 节点依赖关系管理
- 执行日志实时显示

#### 2. 商户信息获取
- `POST /api/workflow/merchant/<mid>` - 自动采集不存在商户
- `POST /api/workflow/website/<mid>` - 获取官网
- `PUT /api/workflow/website/<mid>` - 可编辑官网地址

#### 3. 品牌数据采集
- **谷歌关键词** (`/api/workflow/keywords/<mid>`)
  - 自动启动 Chrome 调试模式
  - 调用 `scrape_google_suggest_browser.py` 采集
  - 采集完成后返回关键词列表

- **SEMrush数据** (`/api/workflow/semrush/<mid>`)
  - 检查官网是否存在
  - 启动新窗口运行 `semrush_via_wmx.py`
  - 采集完成后自动保存到数据库
  - 需要外贸侠登录状态

#### 4. 商品数据采集
- **商品列表** (`/api/workflow/products/<mid>`)
  - 无数据时启动 `download_only.py --single <mid>` 下载
  - 商品下载需要 YP 登录状态

- **亚马逊商品信息** (`/api/workflow/amazon/<mid>`)
  - 添加商品选择下拉框
  - 用户可选择单个商品 ASIN 进行采集
  - 保存用户选择的 ASIN 到 `workflowState.selectedAsin`

#### 5. 广告生成与评估
- **商品报告** (`/api/workflow/report/<mid>`) - 使用用户选择的 ASIN
- **广告方案** (`/api/workflow/ads/<mid>`) - 使用用户选择的 ASIN
- **质量评估** (`/api/workflow/evaluate/<mid>`) - 使用用户选择的 ASIN

#### 6. 状态检查 API
- `GET /api/workflow/status/<mid>` - 检查商户所有已有数据状态
- 开始流程时自动加载已有数据，标记已完成节点

### Bug 修复

1. **节点解锁问题**
   - 问题：`workflowState` 中 `keywords: []`、`products: []`、`amazon: {}` 被误判为 truthy
   - 解决：改为 `null`

2. **SEMrush 采集窗口标题问题**
   - 问题：Windows `start` 命令窗口标题解析错误
   - 解决：使用空字符串 `""` 作为窗口标题

3. **SEMrush 数据未保存**
   - 问题：采集完成后数据未保存到数据库
   - 解决：在 `semrush_via_wmx.py` 中添加调用 `/api/save_semrush_data` API

4. **报告节点未解锁**
   - 问题：`report` 依赖 `['products', 'amazon', 'keywords', 'semrush']`
   - 解决：改为只依赖 `['products', 'amazon']`

5. **下载报告按钮错误**
   - 问题：使用 `merchantId` 而非 `selectedAsin`
   - 解决：改为使用 `workflowState.selectedAsin`
